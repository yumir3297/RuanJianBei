# 设计背景总结 — 供外部设计审查

> 请阅读完本文后，从专业 UI/UX 设计角度给出意见。特别关注：功能布局、信息层级、用户动线、以及是否还有"AI 设计味"的残留。

---

## 一、项目本质

这是一个 **景区导览 AI 数字人** 的 Web 前端（Vue 3 + Vite），面向竞赛评委演示。景区为无锡灵山胜境（佛教文化主题，核心景点：灵山大佛、梵宫、九龙灌浴、五印坛城）。

**硬指标约束：**
- 问答准确率 ≥ 90%（所有回答基于官方资料库，必须可追溯来源）
- 语音问答端到端延迟 < 5 秒
- 数字人具备语音播报、口型同步、状态表情联动
- 支持文本 / 语音 / 图片识别三种输入
- 支持主动选择景点、话题、路线
- 管理后台涵盖知识库、盲区处理、数据大屏、数字人配置

## 二、我们的设计探索过程

### 第一轮：AI 直接生成
效果图华丽但同质化严重——多层渐变、毛玻璃、大圆角卡片、发光边框、彩色装饰条、英文小标签 "SCENIC INTELLIGENCE" 等。用户反馈"太复杂花哨"、"AI味太重"。

### 第二轮：刻意极简
去掉所有装饰，纯白底 + 细线边框。但数字人区域太小（头像 60-80px 放在标题行），用户反馈"数字人导游没存在感"。

### 第三轮：左大数字人 + 右对话 + 底证据
这是当前锁定的方向。具体见下方。

### 设计决策及理由

| 决策 | 理由 |
|------|------|
| 左栏 320px 固定宽 | 给数字人足够的展示空间（Live2D 模型约 180×220px + 状态指示灯 + 波形条） |
| 右栏弹性 | 对话是主体交互区，需要足够宽度承载长文本回复 |
| 底条放证据来源 | 赛题要求"可追溯的检索来源"，但不想让来源信息抢占对话空间 |
| 配色：白底 + 青绿强调 + 浅灰边框 | 灵山是佛教景区，青绿（teal）文化适配；白底高对比度适合评委演示 |
| 只用细线边框，不用阴影 | 阴影在投影仪/屏幕上经常不可见，且是 AI 生成的高频特征 |
| 圆角 ≤ 12px | 圆角过大（16-20px）是 AI 设计的标志性特征 |
| 无渐变背景、无装饰伪元素 | 同上，是去 AI 味的核心手段 |
| 无英文标签 | 景区用户不需要英文 |

### 色彩变量

## 四、我们的自我质疑（请你帮忙判断）

1. **左栏 320px 会不会太宽？** 数字人区填满后还好，但 GoalSelector（景点/话题/路线选择）在左栏竖排展示，如果内容少会显得空洞。有没有更好的组织方式？

2. **底条证据来源**——这种处理方式是一个横跨全宽的窄条（约 40px 高），显示来源卡片。会不会太低、容易被忽略？有什么更好的证据呈现方式？

3. **输入区在对话框底部**（而不是全页面底部），意味着语音/拍照/发送按钮只属于右栏。左栏完全是被动的展示区。这种分区合理吗？

4. **整体还有没有"AI 设计味"的残留？** 我们已经删了渐变、阴影、圆环、英文标签，但可能还有一些不自觉的 AI 味特征。

5. **信息层级**：游客端页面里同时有数字人、对话、目标选择器、输入区、证据来源——五块内容。优先级是否清晰？

## 五、给你参考的真实产品对标

我们在设计过程中研究了这些产品：
- **豆包 / Kimi**：白底为主，头像小且安静，对话是主体
- **Linear**：暗色侧栏 + 白内容区，无多余装饰
- **Apple HIG**：每屏一个任务，户外可读性（大字号、高对比度）
- **ChatGPT / Claude**：对话即界面，来源引用在消息底部
