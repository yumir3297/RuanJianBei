## 一、结论：三个形象是同一个模型

```
小诗 (shizuku) ──┐
                  ├──→ haru_greeter_t03.model3.json  ← Haru 打招呼模型
小响 (hibiki)  ──┤
                  │
小春 (haru)    ──┘

结论：管理层看到 3 张卡片、3 个名字、3 种不同颜色的圆形头像，
      但 Live2D 预览区永远是同一只 Haru 角色。
```

| 卡片名称 | CDN 上实际有不同模型吗 | CDN 实测结果 |

## 五、额外隐患：Cubism 版本兼容

当前导入路径：
```javascript
import { Live2DModel } from "pixi-live2d-display/cubism4";
// cubism4 映射到 dist/cubism4.es.js —— 仅含 Cubism 4 Core
```

而 xiazeyu CDN 上的 Shizuku 和 Hibiki 都是 **Cubism 2.1** 格式（`.moc` + `.model.json`），不是 Cubism 4 格式（`.moc3` + `.model3.json`）。

**cubism4 能否加载 Cubism 2.1 模型？** 从 guansss CDN 的 Shizuku（同样是 Cubism 2.1）能在当前代码中加载来看，`cubism4.es.js` 可能内置了向后兼容。但 xiazeyu CDN 的模型文件结构与 guansss 不同（xiazeyu 使用 `moc/` 子目录前缀），可能存在路径解析问题。

**更安全的做法**：改用基础导入
```javascript
import { Live2DModel } from "pixi-live2d-display";
// 基础导入包含 Cubism 2 + 4 双版本 Core
```

## 六、形象切换链路实际运行时发生了什么

以用户点击 "小响" 卡片为例：

```
1. AvatarConfig.vue  selectModel("hibiki")
                     selectedModel = "hibiki" ✅
                     ↓
2.                  previewAvatarRef.switchModel("hibiki")
                     ↓
3. Live2DAvatar.vue  handleSwitchModel("hibiki")
                     → loadModel("hibiki")
                     ↓
4. useModelManager.js switchModel("hibiki", ...)
                     config = AVATAR_MODELS["hibiki"]
                     config.url = CUBISM4_MODEL_URL  ← = haru URL！
                     ↓
5.                  Live2DModel.from(CUBISM4_MODEL_URL)
                     ↓
6.                  加载 haru_greeter_t03.moc3 + haru 纹理
                     ↓
               → 预览区显示：Haru（不是 Hibiki）
               → 用户看起来：切走了，又切回来了，好像没变化
```

**唯一的不同**：`defaultScale` 分别是 0.18 / 0.17 / 0.17，导致切换时模型有微小缩放跳动，进一步制造"好像变了点"的错觉。

## 八、修复建议

只需改 `useModelManager.js` 一个文件：

```javascript
// 删除第 3-4 行的 CUBISM4_MODEL_URL 常量
// 改为每个模型独立 URL：

export const AVATAR_MODELS = {
  shizuku: {
    key: "shizuku",
    label: "小诗",
    description: "轻快亲和，适合游客导览",
    url: "https://cdn.jsdelivr.net/gh/guansss/pixi-live2d-display@master/test/assets/shizuku/shizuku.model.json",
    defaultScale: 0.18,
  },
  hibiki: {
    key: "hibiki",
    label: "小响",
    description: "稳重知性，适合讲解内容较多的场景",
    url: "https://cdn.jsdelivr.net/gh/xiazeyu/live2d-widget-models@master/packages/live2d-widget-model-hibiki/assets/hibiki.model.json",
    defaultScale: 0.16,
  },
  haru: {
    key: "haru",
    label: "小春",
    description: "明快自然，适合景区迎宾展示",
    url: "https://cdn.jsdelivr.net/gh/guansss/pixi-live2d-display@master/test/assets/haru/haru_greeter_t03.model3.json",
    defaultScale: 0.17,
  },
};
```

**风险提示**：Hibiki 的 xiazeyu CDN 是 Cubism 2.1 格式，`cubism4` 导入可能无法加载。如果切换后 Hibiki 报错，同时做 Step 0（导入路径从 `cubism4` 改为 `pixi-live2d-display`）。

## 九、附：CDN 拉取的三个 model 文件摘要对照

| 模型 | 格式 | 骨骼 | 纹理数 | 表情 | Motion |
|------|------|------|:--:|------|--------|
| **Haru** (当前 3 个都用这个) | Cubism 4 `Version:3` | `haru_greeter_t03.moc3` | 2 | f00-f07 | Idle, Tap |
| **Shizuku** (guansss) | Cubism 2.1 `type:Live2D Model Setting` | `shizuku.moc` | 6 | f01-f04 | idle, tap, shake, flick_head, pinch |
| **Hibiki** (xiazeyu) | Cubism 2.1 `version:Sample 1.0.0` | `hibiki.moc` | 1 | f01-f06 | null, idle |

三个是完全不同的人物、不同的骨骼网格、不同的纹理贴图、不同的动画面。只是代码指向了同一个 URL。
