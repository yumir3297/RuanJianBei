## 二、问题 1：大模型编造事实

### 2.1 问题定位

LLM 在缺乏证据时倾向于编造看似合理的信息。对于景区导览场景，编造的后果是游客获得错误信息（虚假的票价、不存在的景点、编造的历史事件）。

### 2.2 技术方案：Prompt 强制证据链 + 答案审计 + 来源引用

#### 2.2.2 答案合规校验器

```python
# backend/app/services/qa/answer_validator.py

import re
from app.services.rag.base import RetrievedDocument


class AnswerValidator:
    """
    回答后校验器：检查 LLM 生成的答案是否与检索证据一致。
    不阻塞回答流式输出，异步运行，结果写入 ChatLog 供审计。
    """
    MIN_KEYWORD_MATCH_RATIO = 0.3
    HALLUCINATION_PATTERNS = [
        r"\d+元",                  # 疑似编造价格
        r"\d{4}年\d{1,2}月\d{1,2}日",  # 具体日期
        r"每天\d{1,2}:\d{2}",       # 具体时间
        r"门票.*\d+",               # 门票数字
    ]

    def validate(
        self,
        answer: str,
        documents: list[RetrievedDocument],
    ) -> dict:
        combined_text = " ".join(doc.content for doc in documents)
        answer_words = set(re.findall(r"[\u4e00-\u9fff\w]+", answer.lower()))
        source_words = set(re.findall(r"[\u4e00-\u9fff\w]+", combined_text.lower()))

        if not answer_words:
            return {"compliant": False, "reason": "empty_answer"}

        if not source_words:
            return {"compliant": True, "reason": "no_documents_provided"}

        match_ratio = len(answer_words & source_words) / max(len(answer_words), 1)
        compliant = match_ratio >= self.MIN_KEYWORD_MATCH_RATIO

        suspicious_matches = []
        for pattern in self.HALLUCINATION_PATTERNS:
            matches = re.findall(pattern, answer)
            if matches:
                for m in matches:
                    if m not in combined_text:
                        suspicious_matches.append({"pattern": pattern, "matched": m})

        return {
            "compliant": compliant,
            "keyword_match_ratio": round(match_ratio, 3),
            "suspicious_hallucinations": suspicious_matches,
            "verdict": "pass" if compliant and not suspicious_matches else "review",
        }
```

## 三、问题 2：户外噪音 / 方言影响语音识别

### 3.1 问题定位

景区户外场景存在环境噪音（风声、人声、背景音乐），且游客可能使用吴语、粤语等方言。ASR 识别错误将导致后续 FAQ/RAG 全链路失效。

### 3.2 技术方案：ASR 置信度降级 + LLM 方言改写 + 手动选择

```
┌──────────────────────────────────────────────────────┐
│  语音输入逻辑                                         │
│                                                      │
│  ASR Result {text, confidence}                       │
│       │                                              │
│       ├── confidence ≥ 0.8  → 直接使用 ASR 文本       │
│       │                                              │
│       ├── 0.5 ≤ confidence < 0.8                     │
│       │   → LLM 方言改写：纠正发音同音词、方言词       │
│       │                                              │
│       └── confidence < 0.5                           │
│           → 前端展示 ASR 文本供游客确认修改            │
│           → 同时推送"试试文字输入"提示                 │
│           → 推送"选择您所在的景点"快捷入口             │
└──────────────────────────────────────────────────────┘
```

```python
# backend/app/services/asr/dialect_rewriter.py

DIALECT_REWRITE_PROMPT = """将以下可能包含方言发音误识别或口语表达的景区问询改写为规范普通话。
常见误识别纠正示例：
- "零三发佛" → "灵山大佛"（吴语发音）
- "个只大佛" → "这个大佛"
- "几钿一张票" → "一张票多少钱"
- "高头" → "高度"

规则：
1. 只输出改写后的规范问题，不要解释
2. 如果已经是规范普通话，原样返回
3. 保留语义，不要改变问题意图

输入：{query}
输出："""


class DialectQueryRewriter:
    def __init__(self, llm_service):
        self.llm_service = llm_service
        self.prompt_template = DIALECT_REWRITE_PROMPT

    async def rewrite(self, query: str) -> str:
        prompt = self.prompt_template.format(query=query)
        result = []
        async for token in self.llm_service.stream_generate(
            prompt, documents=[]
        ):
            result.append(token)
        rewritten = "".join(result).strip()
        if not rewritten:
            return query
        # 防 LLM 偏离：超长输出截断
        return rewritten if len(rewritten) <= 200 else query
```

## 四、问题 3：多轮对话上下文丢失

### 4.1 问题定位

当前 `ConversationContext` 仅含 `last_subject` 和 `history_summary`。`QueryRewriter.rewrite()` 只在"缺主语 + 提问词开头"时补 `last_subject`。对于"那附近还有什么""有优惠吗"等半命题式追问，现有方案无法处理。

### 4.2 技术方案：会话上下文管理器 + LLM 指代消解 + 实体追踪

```text
示例对话：
  Q1: 灵山大佛有多高
  A1: 通高88米...
  Q2: 它是纯铜的吗 → 指代消解 → "灵山大佛是纯铜的吗"
  Q3: 那附近有什么景点 → 指代消解 → "灵山大佛附近有什么景点"
  Q4: 怎么坐车去 → 指代消解 → "怎么坐车去灵山大佛"
```

#### 4.2.1 核心实现

```python
# backend/app/services/qa/context_manager.py

from collections import deque, defaultdict
from datetime import datetime


class ConversationTurn:
    __slots__ = ("query", "answer", "entities", "timestamp")
    def __init__(self, query: str, answer: str = "", entities: list[str] | None = None):
        self.query = query
        self.answer = answer
        self.entities = entities or []
        self.timestamp = datetime.now()


class ConversationContextManager:
    """
    维护 session 级对话历史，提供指代消解能力。
    每个 session 最多保留 10 轮对话。
    """
    REFERENCE_WORDS = frozenset({
        "它", "他", "她", "那", "这个", "那个",
        "这里", "那里", "这", "那儿", "那边",
        "这边", "这样", "那种",
    })

    def __init__(self, max_turns: int = 10):
        self.max_turns = max_turns
        self.history: dict[str, deque[ConversationTurn]] = defaultdict(
            lambda: deque(maxlen=max_turns)
        )
        self.entity_tracker: dict[str, dict[str, int]] = defaultdict(
            lambda: defaultdict(int)
        )

    def add_turn(
        self,
        session_id: str,
        query: str,
        answer: str,
        entities: list[str] | None = None,
    ) -> None:
        turn = ConversationTurn(query=query, answer=answer, entities=entities)
        self.history[session_id].append(turn)
        for entity in (entities or []):
            self.entity_tracker[session_id][entity] += 1

    def needs_reference_resolution(self, session_id: str, query: str) -> bool:
        """判断是否需要指代消解"""
        if not any(word in query for word in self.REFERENCE_WORDS):
            return False
        return len(self.history[session_id]) > 0

    def get_reference_context(self, session_id: str) -> str:
        """构建指代消解的对话历史上下文"""
        turns = list(self.history[session_id])[-3:]
        lines = []
        for t in turns:
            lines.append(f"游客：{t.query}")
            if t.answer:
                lines.append(f"系统：{t.answer[:200]}")
        return "\n".join(lines)

    def get_active_entities(self, session_id: str, top_n: int = 5) -> list[str]:
        """获取当前会话中的高频实体"""
        entities = self.entity_tracker[session_id]
        sorted_entities = sorted(entities.items(), key=lambda x: x[1], reverse=True)
        return [e for e, _ in sorted_entities[:top_n]]
```

#### 4.2.2 LLM 指代消解

```python
# backend/app/services/qa/context_manager.py (续)

REFERENCE_RESOLVE_PROMPT = """根据对话历史，将当前问题中的指代词替换为具体实体。

对话历史：
{history}

当前问题（包含指代词）：
{query}

请将当前问题中的"它""那""这个""那里"等指代词替换为具体的景点名、事物名或地点名。
只输出改写后的问题，不要解释，不要加引号。

输出："""


class ContextManagerService:
    def __init__(self, llm_service):
        self.manager = ConversationContextManager()
        self.llm_service = llm_service

    async def resolve_query(
        self, session_id: str, query: str
    ) -> dict:
        result = {"query": query, "did_resolve": False}

        if not self.manager.needs_reference_resolution(session_id, query):
            return result

        context = self.manager.get_reference_context(session_id)
        prompt = REFERENCE_RESOLVE_PROMPT.format(history=context, query=query)

        resolved_parts = []
        async for token in self.llm_service.stream_generate(prompt, documents=[]):
            resolved_parts.append(token)
        resolved = "".join(resolved_parts).strip()

        if resolved and resolved != query and len(resolved) <= 200:
            result["query"] = resolved
            result["did_resolve"] = True

        return result
```

## 六、问题 5：节假日高并发压力

### 6.1 问题定位

#### 6.2.1 FAQ L3 语义匹配

```python
# backend/app/services/qa/faq_matcher.py — 补完

def match_semantic(self, query: str, embedding_fn=None) -> FAQEntry | None:
    if embedding_fn is None:
        return None
    if not self.entries:
        return None

    query_vec = embedding_fn(query)

    best_score = 0.0
    best_entry = None
    SIMILARITY_THRESHOLD = 0.85

    for entry in self.entries:
        for alias in entry.aliases:
            alias_vec = embedding_fn(alias)
            score = self._cosine_similarity(query_vec, alias_vec)
            if score > best_score:
                best_score = score
                best_entry = entry

    if best_score >= SIMILARITY_THRESHOLD and best_entry:
        return best_entry
    return None
```

性能优化：对 FAQ 的所有 alias 预先向量化并缓存，避免每次匹配都重新做 embedding。

```python
def precompute_semantic_index(self, embedding_fn) -> None:
    """启动时预计算所有 FAQ alias 的向量"""
    self.semantic_index: dict[str, list[float]] = {}
    for entry in self.entries:
        for alias in entry.aliases:
            self.semantic_index[alias] = embedding_fn(alias)
```

```python
# backend/app/services/qa/concurrency.py

import asyncio


class LLMConcurrencyLimiter:
    def __init__(self, max_concurrent: int = 5):
        self.semaphore = asyncio.Semaphore(max_concurrent)
        self.max_concurrent = max_concurrent

    async def generate(self, llm_service, query, documents):
        async with self.semaphore:
            async for token in llm_service.stream_generate(query, documents):
                yield token

    async def generate_with_timeout(
        self, llm_service, query, documents, timeout: float = 30.0
    ):
        async with self.semaphore:
            try:
                async with asyncio.timeout(timeout):
                    async for token in llm_service.stream_generate(query, documents):
                        yield token
            except asyncio.TimeoutError:
                yield "\n\n[回答时间较长，已为您精简结果。如需更多信息，请重新提问。]"
```

```python
# backend/app/services/sentiment/analyzer.py

from snownlp import SnowNLP
from dataclasses import dataclass


@dataclass(slots=True)
class SentimentResult:
    score: float
    label: str

    def to_dict(self) -> dict:
        return {"score": self.score, "label": self.label}


class SentimentAnalyzer:
    def analyze(self, text: str) -> SentimentResult:
        if not text.strip():
            return SentimentResult(score=0.5, label="neutral")
        s = SnowNLP(text)
        score = s.sentiments
        if score > 0.7:
            label = "positive"
        elif score < 0.3:
            label = "negative"
        else:
            label = "neutral"
        return SentimentResult(score=score, label=label)
```

### 8.5 后端引导逻辑

```python
# backend/app/services/qa/guided_selector.py

from dataclasses import dataclass


@dataclass
class SelectionHint:
    topic_key: str | None = None
    spot_key: str | None = None
    search_boost: str | None = None
    prompt_hint: str | None = None
    trigger_recommend: bool = False


class GuidedSelector:
    """
    根据游客的前端选择，返回检索增强指令和回答引导指令。
    """

    def __init__(self, topic_repo, spot_repo):
        self.topic_repo = topic_repo
        self.spot_repo = spot_repo

    def resolve(
        self,
        topic_key: str | None = None,
        spot_key: str | None = None,
    ) -> SelectionHint:
        hint = SelectionHint()

        if spot_key:
            spot = self.spot_repo.get_by_key(spot_key)
            if spot:
                hint.spot_key = spot_key
                hint.search_boost = spot.search_hint
                hint.prompt_hint = spot.prompt_hint

        if topic_key:
            topic = self.topic_repo.get_by_key(topic_key)
            if topic:
                hint.topic_key = topic_key
                if topic.search_hint:
                    # topic 的 search_hint 可以是特殊信号
                    if topic.search_hint == "knowledge_entries":
                        hint.search_boost = (
                            hint.search_boost or ""
                        )
                    elif topic.search_hint == "faq":
                        hint.search_boost = "faq"
                    elif topic.search_hint == "recommend":
                        hint.trigger_recommend = True
                    else:
                        hint.search_boost = (
                            f"{hint.search_boost or ''} {topic.search_hint}".strip()
                            or topic.search_hint
                        )
                if topic.prompt_hint:
                    hint.prompt_hint = (
                        f"{hint.prompt_hint or ''}。{topic.prompt_hint}".strip("。")
                    )

        return hint
```

## 十、执行优先级与阶段规划

```
阶段          内容                                  依赖           得分影响
──────────────────────────────────────────────────────────────────────────
Phase A     LLM 真实化 + 证据链 Prompt (问题1)       无              准确率核心
Phase A     并发控制 + FAQ L3 + Chroma预热 (问题5)    bge-small      延迟核心
Phase B     会话上下文 + 指代消解 (问题3)             LLM真实化      体验完整性
Phase B     游客主动选择引导 (问题7)                   独立            体验加分
Phase C     知识盲区追踪 (问题4)                       独立            功能完整度
Phase C     情感分析 + 人格化话术 (问题6)              SnowNLP        创新加分
Phase D     ASR 真实化 + 方言改写 (问题2)              阿里云NLS       体验性
```

### 执行原则

1. **Phase A 必须先做**：LLM 真实化 + 并发控制 + FAQ L3 是准确率和延迟两个硬指标的基石
2. **Phase B/C 可并行**：会话上下文、游客选择、盲区追踪、情感分析之间无相互阻塞
3. **Phase D 可延后**：ASR 方言需要外部 API 账号和费用审批，可在主链路稳定后追加
4. **每阶段完成后运行现有 20 个测试 + 30 题检索评测**，确认不退化

## 十一、风险与注意事项

| 风险 | 应对 |
|------|------|
| LLM API 不可用/额度不足 | 保留 FAQ + Cache 独立工作能力；LLM 不可用时长链路退化为 FAQ 模式 |
| SnowNLP 对短句式判断不准 | `score > 0.7 / < 0.3` 阈值保守设置，中性区间宽；ChatLog 记录原始分数供调参 |
| LLM 指代消解增加延迟 | 仅检测到指代词时才触发；超时 1s 回退为无消解 query |
| QuickTopic/Spot 数据与知识库不同步 | bootstrap 仅写预设，管理端可编辑；`search_hint` 为软关联，不阻塞 |
| 并发限制器导致排队超时 | 设置 30s 总体超时 + 超时后返回精简结果 |
