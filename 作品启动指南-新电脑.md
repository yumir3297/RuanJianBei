# 作品在其他电脑上运行指南（从零开始）

> 这份文档是给需要在**自己电脑上**运行作品的组员的。按照步骤操作即可，不需要编程基础。

---

## 一、他需要从你的电脑上拿什么

把你的整个项目文件夹 `D:\桌面\软件杯` 拷给他。建议用移动硬盘或压缩后网盘传。

**但以下 3 个文件夹不用拷**（太大了，而且在他的电脑上需要重新生成）：

```
❌ 不要拷  backend\.venv        (Python环境，拷了也用不了)
❌ 不要拷  backend\__pycache__  (Python缓存)
❌ 不要拷  frontend\node_modules (前端依赖，拷了也用不了)
```

**以下都必须拷**：

```
✅ 全部的子文件夹和文件，尤其是：
   ├── backend\app\           (所有后端代码)
   ├── backend\main.py
   ├── backend\requirements.txt
   ├── backend\.env.example
   ├── frontend\src\          (所有前端代码)
   ├── frontend\package.json
   ├── frontend\vite.config.js
   ├── data\processed\        (景区资料，约 500KB)
   ├── kb\chroma_db\          (向量检索索引)
   ├── .cache\huggingface\    (AI 模型文件，约 1.5GB)
   └── docs\作品启动指南.md
```

拷完后，假设他把项目放在 `D:\软件杯`（或其他任何路径，路径可以不同）。

## 二、他需要在电脑上安装什么

### 2.1 安装 Python

1. 打开浏览器，访问 **https://www.python.org/downloads/**
2. 点击黄色的大按钮 "Download Python 3.13.x"
3. 运行下载的安装程序
4. **重要**：第一屏勾选 ☑ **Add Python to PATH**（底部那个复选框）
5. 点击 "Install Now"
6. 等安装完成

验证是否装好：打开 PowerShell（开始菜单搜"PowerShell"），输入：

```powershell
python --version
```

应该显示 `Python 3.13.x`

### 2.2 安装 Node.js

1. 打开浏览器，访问 **https://nodejs.org/**
2. 下载左边的 "LTS" 版本（推荐）
3. 运行安装程序，一路 "Next" 即可

验证是否装好：

```powershell
node --version
```

应该显示类似 `v20.x.x` 或 `v22.x.x`

## 三、一次性环境配置（只需做一次，约 10 分钟）

> 以下所有操作都在 **PowerShell** 中进行。
> 把下面命令里的 `D:\软件杯` 替换成他实际放项目的位置。

### 3.1 创建 Python 虚拟环境 + 安装依赖

```powershell
cd D:\软件杯\backend

# 创建虚拟环境（只需做一次）
python -m venv .venv

# 激活虚拟环境
.\.venv\Scripts\activate

# 先装 CPU 版 torch（体积小，下载快）
pip install torch --index-url https://download.pytorch.org/whl/cpu

# 再装其他依赖
pip install -r requirements.txt
```

**这一步需要联网。** 大约下载 500MB，根据网速需要 3-10 分钟。

### 3.2 安装前端依赖

```powershell
cd D:\软件杯\frontend
npm install
```

**这一步需要联网。** 大约 1-2 分钟。

### 3.3 创建配置文件

```powershell
cd D:\软件杯\backend

# 复制示例配置文件
copy .env.example .env
```

### 3.4 同步数据（把景区资料导入数据库）

```powershell
cd D:\软件杯\backend
.\.venv\Scripts\python.exe -m uvicorn main:app --host 127.0.0.1 --port 8000
```

等看到启动成功的提示后，**再开一个 PowerShell 窗口**：

```powershell
# 这个操作会把景区知识、FAQ、路线等都导入到数据库
curl -X POST http://127.0.0.1:8000/api/admin/sync-processed-data
```

应该看到类似这样的输出：
```json
{"message":"Processed data synced successfully.","report":{"knowledge_imported":38,"chunk_imported":66,"faq_imported":88,"route_imported":3,...}}
```

### 3.5 构建 AI 向量索引

```powershell
# 这一步会用 AI 模型把所有知识"向量化"，让系统能智能检索
curl -X POST http://127.0.0.1:8000/api/admin/build-rag-index
```

应该看到类似这样的输出：
```json
{"message":"RAG index built successfully.","report":{"total_chunks":66,"indexed_chunks":66,...}}
```

**如果模型已经拷过来了**（`.cache\huggingface` 里有文件），这步很快，约 10 秒。
**如果模型没拷过来**，这步会自动从网上下载（约 500MB），需要几分钟。

### 3.6 关掉后端窗口

按 `Ctrl + C` 关闭两个后端窗口。

## 四、日常启动（每次演示只需 2 分钟）

### 第一步：启动后端

打开 PowerShell：

```powershell
cd D:\软件杯\backend
.\.venv\Scripts\python.exe -m uvicorn main:app --host 127.0.0.1 --port 8000
```

看到 `Uvicorn running on http://127.0.0.1:8000` 就是启动了。

**这个窗口不要关。**

### 第二步：启动前端

**再开一个** PowerShell 窗口：

```powershell
cd D:\软件杯\frontend
npm run dev
```

看到 `Local: http://localhost:5173/` 就是启动了。

### 第三步：打开浏览器

浏览器输入 `http://localhost:5173/` → 看到游客端的页面。

| 安装 Python / Node.js | ✅ 需要 |

## 六、常见问题

### "pip install 报错 / 下载太慢"

换成国内镜像：

```powershell
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### "端口被占用"

后端默认用 8000 端口，前端用 5173。如果提示端口被占用，先把之前开的窗口关掉再试。

### "页面打开了但是空白"

1. 确认后端窗口还在运行（没有报错）
2. 确认前端窗口还在运行
3. 按 F5 刷新浏览器

### "模型下载太慢或失败"

这就是为什么要拷 `.cache\huggingface` 文件夹。如果拷过来了，系统会自动用本地的，不会从网上下载。

模型文件在：
```
D:\软件杯\.cache\huggingface\models--BAAI--bge-small-zh-v1.5\
D:\软件杯\.cache\huggingface\models--BAAI--bge-reranker-base\
```

只要这两个文件夹（及其子文件夹）都存在，就不需要联网下载模型。

## 七、整个准备流程总结

```
拷贝项目（不需要 .venv / node_modules / __pycache__）
    │
安装 Python 3.13 + Node.js
    │
pip install 后端依赖（首次需要联网）
    │
npm install 前端依赖（首次需要联网）
    │
创建 .env 配置文件
    │
启动后端 → sync-processed-data → build-rag-index → 关掉
    │
以后每次：启动后端 → 启动前端 → 打开浏览器 → 开始演示
```
