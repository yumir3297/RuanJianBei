## 二、改进方案

### 方案A：口型自然化（立即实施）

#### 问题
当前代码（Live2DAvatar.vue第84-91行）：
```javascript
mouthTimer = setInterval(() => {
  open = (open + 1) % 2;
  model.internalModel.coreModel.setParameterValueById("ParamMouthOpenY", open ? 0.8 : 0.0);
}, 150);
```
- 固定150ms切换一次
- 只有0和0.8两个状态
- 没有过渡，过于生硬

#### 改进
1. **添加随机变化**：模拟真实说话的口型变化
2. **平滑过渡**：使用渐变动画而非突变
3. **可变频率**：根据内容长度调整节奏

**修改代码**：
```javascript
function startMouthAnimation() {
  if (mouthTimer) return;
  
  let currentOpen = 0;
  let targetOpen = 0;
  let frameCount = 0;
  
  mouthTimer = setInterval(() => {
    if (!model) return;
    
    // 每8-12帧切换一次目标值（模拟说话节奏）
    if (frameCount % Math.floor(Math.random() * 5 + 8) === 0) {
      targetOpen = Math.random() * 0.6 + 0.2; // 0.2-0.8之间随机
    }
    
    // 平滑过渡到目标值
    currentOpen += (targetOpen - currentOpen) * 0.3;
    
    try {
      model.internalModel.coreModel.setParameterValueById("ParamMouthOpenY", currentOpen);
    } catch {}
    
    frameCount++;
  }, 50); // 更高频率的更新（20fps）
}
```

**优势**：
- 口型变化更自然，有大有小
- 平滑过渡，无突变感
- 模拟真实说话节奏

function playMotion(motionName) {

  }

### 方案C：音频驱动口型（进阶，可选）

如果时间充裕，可以实现真正的音频驱动：

1. **使用Web Audio API分析音频**
2. **提取音量和频率**
3. **实时映射到口型参数**

**代码示例**：
```javascript
// 分析TTS音频流
function analyzeAudioStream(audioElement) {
  const audioContext = new AudioContext();
  const source = audioContext.createMediaElementSource(audioElement);
  const analyser = audioContext.createAnalyser();
  
  source.connect(analyser);
  analyser.connect(audioContext.destination);
  
  const dataArray = new Uint8Array(analyser.frequencyBinCount);
  
  function updateMouth() {
    analyser.getByteFrequencyData(dataArray);
    const volume = dataArray.reduce((a, b) => a + b) / dataArray.length;
    const mouthOpen = Math.min(volume / 128, 1.0); // 归一化到0-1
    
    model.internalModel.coreModel.setParameterValueById("ParamMouthOpenY", mouthOpen);
    requestAnimationFrame(updateMouth);
  }
  
  updateMouth();
}
```

**但注意**：
- 需要系统有TTS语音输出
- 增加复杂度
- 如果没有语音输出，此方案不适用
