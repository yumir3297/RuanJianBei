# 产品部署和可执行手册

## 第1章 产品概述

### 1.1 产品名称与版本号
灵山胜境 AI 数字人导览系统 v1.0

### 1.2 产品简介
面向灵山胜境景区的 AI 数字人导览系统，游客端支持文本、语音、图片和主动选择多模态交互，后台支持知识管理、数字人配置、游客分析和数据大屏。核心知识基于景区官方资料的 RAG 证据回答。

### 1.3 适用场景与用户群体
- 游客端：灵山胜境景区游客，通过 Web 页面访问
- 管理后台：景区运营管理人员，通过 Web 页面管理

### 1.4 名词术语表
| 术语 | 说明 |
|------|------|
| RAG | 检索增强生成，先从知识库检索相关资料，再交给大模型生成回答 |
| ChromaDB | 向量数据库，用于存储和检索知识片段 |
| SSE | Server-Sent Events，服务器向浏览器推送流式数据 |
| Embedding | 将文本转换为向量表示，用于语义检索 |
| CrossEncoder / Reranker | 对候选检索结果进行精排重排序 |
| FAQ L3 | 基于语义向量的 FAQ 匹配，预计算后毫秒级响应 |
| ASR | 自动语音识别 |
| TTS | 文本转语音 |
| SelectionContext | 游客主动选择的景点/话题/路线上下文结构 |

## 第2章 运行环境要求

### 2.1 硬件要求
- CPU：4 核及以上
- 内存：8GB 及以上（推荐 16GB）
- 磁盘：建议留出 D 盘 10GB+ 空间用于存储 Embedding 模型和 ChromaDB 索引
- 网络：可访问 DeepSeek API 和阿里云百炼 API

### 2.2 操作系统
- Windows 10/11（推荐）
- Linux（Ubuntu 20.04+）

### 2.3 软件依赖
| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.10+ | 后端运行环境 |
| Node.js | 18+ | 前端构建环境 |
| Git | 2.x | 版本控制 |
| SQLite | 内置 | 数据库，无需独立安装 |
| ChromaDB | pip 安装 | 向量数据库 |

### 2.4 第三方服务账号
| 服务 | 用途 | 获取方式 |
|------|------|---------|
| DeepSeek API Key | 大模型文本问答 | https://platform.deepseek.com |
| 阿里云百炼 API Key | Qwen 图片识别 | https://bailian.console.aliyun.com |
| （可选）ASR/TTS Key | 语音识别与合成 | 待确认 Provider |

## 第3章 快速部署

### 3.1 获取源码
```bash
git clone <仓库地址>
cd 软件杯
```
或解压源码包至工作目录。

### 3.2 后端部署

#### 3.2.1 创建虚拟环境
```bash
cd backend
python -m venv venv
```

#### 3.2.2 激活虚拟环境
Windows:
```bash
venv\Scripts\activate
```
Linux:
```bash
source venv/bin/activate
```

#### 3.2.3 安装依赖
```bash
pip install -r requirements.txt
```

#### 3.2.4 配置环境变量
```bash
copy .env.example .env
```
编辑 `.env`，填入以下必填项：
```
DEEPSEEK_API_KEY=your_deepseek_api_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

VISION_PROVIDER=qwen
VISION_API_KEY=your_qwen_api_key
VISION_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
VISION_MODEL=qwen-vl-plus

HF_HUB_OFFLINE=1
```

#### 3.2.5 初始化数据库与索引
```bash
python -c "from app.db.bootstrap import init_db; init_db()"
```

#### 3.2.6 启动后端
```bash
python main.py
```
默认监听 `http://localhost:8010`。

### 3.3 前端部署

#### 3.3.1 安装依赖
```bash
cd frontend
npm install
```

#### 3.3.2 启动开发服务器
```bash
npm run dev
```
默认监听 `http://localhost:5173`。

#### 3.3.3 生产构建
```bash
npm run build
```
产物在 `frontend/dist/` 目录。

### 3.4 验证部署

#### 3.4.1 后端验证
访问 `http://localhost:8010/docs`，应显示 Swagger API 文档页面。

#### 3.4.2 前端验证
访问 `http://localhost:5173`，应显示游客端页面。

#### 3.4.3 问答链路验证
在前端输入"灵山大佛有多高？"，观察是否有 SSE 流式回答返回。

#### 3.4.4 知识库验证
```bash
curl http://localhost:8010/api/quick-select/bootstrap
```
应返回景点、话题、路线数据。

## 第5章 离线部署（无网络环境）

### 5.1 前提条件
- 所有 Python 依赖已离线下载为 wheel 包
- Embedding 模型已预下载到 D 盘本地路径
- ChromaDB 索引已离线构建完成

### 5.2 离线安装步骤
```bash
pip install --no-index --find-links=./offline_packages -r requirements.txt
```

### 5.3 离线启动确认
```bash
set HF_HUB_OFFLINE=1
python main.py
```
启动日志中应显示 `local_files_only=True`，Embedding 模型从本地加载。

## 第7章 常见问题排查

### 7.1 后端启动报 ModuleNotFoundError
```
原因：Python 依赖未完全安装
解决：pip install -r requirements.txt
```

### 7.2 ChromaDB 索引加载失败
```
原因：kb/chroma_db/ 目录中 chroma.sqlite3 缺失或损坏
解决：重新运行知识库同步脚本重建索引
```

### 7.3 前端请求后端跨域报错
```
原因：后端 CORS 未正确配置或端口不一致
解决：检查 backend/app/core/config.py 中 CORS 设置
     确认前端 .env 中的 VITE_API_BASE 指向 http://localhost:8010
```

### 7.4 Embedding 模型加载失败
```
原因：网络不可用且 HF_HUB_OFFLINE 未设置
解决：设置环境变量 HF_HUB_OFFLINE=1
     确认 D 盘模型路径存在且包含 config.json、pytorch_model.bin 等文件
```

### 7.5 DeepSeek API 调用超时
```
原因：API Key 无效或网络不通
解决：检查 .env 中 DEEPSEEK_API_KEY 是否正确
     确认网络可访问 https://api.deepseek.com
```

### 7.6 端口已被占用
```
原因：8010 或 5173 端口被其他程序占用
解决：
  后端：修改 main.py 中的端口号
  前端：修改 vite.config.js 中的 server.port
```

### 7.7 首次启动知识库为空
```
原因：数据预处理未执行
解决：进入 backend 目录执行数据同步脚本
     python -c "from app.db.bootstrap import sync_knowledge; sync_knowledge()"
```
