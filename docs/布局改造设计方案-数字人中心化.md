## 一、当前布局问题分析

### 现状
```
┌─────────────────────────────────────────────────────┐
│  [小数字人]  标题                         [状态]    │  ← 头部挤在一起
├─────────────────────────────────────────────────────┤
│                                        │            │
│  聊天消息列表（60%宽度）               │ 证据来源   │  ← 左右平铺
│                                        │ （40%）    │
│                                        │            │
├─────────────────────────────────────────────────────┤
│  [功能面板] [输入框]                                 │
└─────────────────────────────────────────────────────┘
```

**问题**：
1. 数字人太小（180x220px），不突出
2. 左右平铺，没有视觉焦点
3. 证据来源和聊天同等重要，但游客只关心对话

## 二、新布局设计（数字人中心化）

### 设计理念
**数字人是导游，应该是视觉中心；对话是核心交互，应该最醒目；其他功能是辅助，可收起。**

### 布局结构
```
┌─────────────────────────────────────────────────────────────────┐
│                         顶部导航栏                               │
├────────────────┬────────────────────────────────────────────────┤
│                │                                                 │
│                │    [您]  游客消息气泡                           │
│                │                                                 │
│   数字人       │    [小灵] 导游回答，带引文标签 [1] [2]         │
│   Live2D       │                                                 │
│   显示区域     │    继续探索面板（追问建议）                     │
│                │                                                 │
│   40%宽度      │    ────────────────────────────────────        │
│   占满高度     │                                                 │
│                │    [游览方式选择]  亲子/休闲/文化深度           │
│                │                                                 │
│                │    输入框：问我关于灵山的任何问题...            │
│                │                                                 │
│                │    [向导游提问] [语音提问]  [📷 拍照识景]      │
│                │                                                 │
├────────────────┴────────────────────────────────────────────────┤
│  [▼ 展开资料来源] （默认收起，点击展开，悬浮在底部）             │
└─────────────────────────────────────────────────────────────────┘
```

### 关键尺寸规格

| 区域 | 尺寸/占比 | 说明 |
|------|----------|------|
| **数字人区域** | 40% 宽度，占满视口高度减去顶栏 | 固定左侧，sticky定位 |
| **对话区域** | 60% 宽度 | 滚动容器，主交互区 |
| **数字人Canvas** | 480px × 720px | 从 180x220 放大到原来的2.7倍 |
| **资料来源** | 收起时：60px高度；展开时：400px | 悬浮抽屉，默认收起 |
| **移动端** | 数字人300px宽，对话占剩余 | 响应式调整 |

## 七、风险与注意事项

### 风险1：数字人加载变慢
**原因**：Canvas尺寸增大，渲染压力增加  
**应对**：
- 保持CDN模型，不用过大的本地模型
- 添加loading骨架屏
- 低配设备自动降低分辨率

### 风险2：移动端数字人太大
**原因**：40%在小屏上仍占太多空间  
**应对**：
- 768px以下改为顶部横向，高度300px
- 可选：移动端隐藏数字人，只保留对话

### 风险3：用户习惯改变
**原因**：左右布局 → 主次布局，需要适应  
**应对**：
- 首次进入显示引导提示
- 资料来源按钮加数字badge提醒
