## 第1章 产品概述

### 1.1 产品名称与版本

- **产品名称：** 灵山胜境 AI 数字人智能导览系统
- **版本号：** v1.0
- **赛事类别：** 软件应用与开发类（A5 赛道）

### 1.2 产品简介

面向灵山胜境景区的 AI 数字人导览系统。游客端支持文本、语音、图片和主动选择的多模态交互；管理后台支持知识管理、数字人配置、游客分析和数据大屏。系统采用 RAG（检索增强生成）架构，所有回答均基于景区官方资料生成并提供来源标注。

### 1.3 系统架构概览

```
浏览器（前端）              FastAPI 后端              外部服务
┌──────────────┐    SSE/    ┌──────────────┐    HTTP    ┌──────────────┐
│ Vue 3 SPA    │◄──────────►│ API / 问答编排 │◄──────────►│ DeepSeek LLM │
│ + Three.js   │   JSON     │              │           ├──────────────┤
│ + VRM 数字人  │           │ RAG / ASR    │           │ 百炼 TTS/ASR │
└──────────────┘           │ TTS / Vision │           ├──────────────┤
                           └──────┬───────┘           │ Qwen Vision  │
                                  │                   └──────────────┘
                           ┌──────┴───────┐
                           │ PostgreSQL   │
                           │ ChromaDB     │
                           └──────────────┘
```

## 第3章 部署与启动步骤

### 3.1 准备工作（一次性，约 20 分钟）

#### 步骤一：安装 Python 3.13+

1. 访问 https://www.python.org/downloads/，下载 Python 3.13.x 安装包
2. 运行安装程序，**务必勾选底部 "Add Python to PATH" 复选框**
3. 验证安装（打开 PowerShell）：
   ```powershell
   python --version
   # 应输出：Python 3.13.x
   ```

#### 步骤二：安装 Node.js

1. 访问 https://nodejs.org/，下载 LTS 版本
2. 运行安装程序，默认选项即可
3. 验证安装：
   ```powershell
   node --version   # 应输出 v20.x.x 或 v22.x.x
   npm --version
   ```

#### 步骤三：安装并配置 PostgreSQL

1. 访问 https://www.postgresql.org/download/windows/ 下载 PostgreSQL 14+ 安装包
2. 安装时记住设置的 **postgres 用户密码**（建议使用 `postgres`）
3. 安装完成后，打开 **pgAdmin** 或命令行，创建数据库：
   ```sql
   CREATE DATABASE a5_scenic_guide;
   ```
   或通过命令行：
   ```powershell
   & "C:\Program Files\PostgreSQL\16\bin\createdb.exe" -U postgres a5_scenic_guide
   ```

#### 步骤四：获取项目源码

将项目文件夹复制到本地（如 `D:\软件杯`）。

## 第5章 后端加载时间说明

后端启动是一个**冷启动**过程，启动时间主要由模型预加载决定。以下为典型耗时（基于 4 核 CPU / 16GB 内存环境实测）：

| 启动阶段 | 耗时 | 说明 |
|---------|:----:|------|
| 数据库迁移（Alembic） | ~2 秒 | 自动执行未完成的迁移 |
| 引导数据检查 | ~1 秒 | 确认快捷话题、数字人配置、示例数据 |
| **Embedding 模型加载** | **~10-20 秒**（首次） / **~5 秒**（二次） | BGE-small-zh-v1.5，约 512 MB，首次从磁盘读取慢 |
| **Reranker 模型加载** | **~15-30 秒**（首次） / **~8 秒**（二次） | BGE-reranker-base，约 1 GB，首次从磁盘读取慢 |
| FAQ 语义索引预热 | ~0.3 秒 | 202 条 FAQ alias 向量编码 |
| **合计冷启动时间** | **约 30-60 秒**（首次） / **约 15 秒**（二次） | 操作系统文件缓存会显著加速二次启动 |

**判断启动完成的标志：** 控制台无报错，且输出 `Uvicorn running on http://127.0.0.1:8000` 之后，建议再等待 5 秒确保所有模型完全加载到内存。

> **注意：** 若 `HF_HUB_OFFLINE=1` 且本地模型缓存不存在，embedding/reranker 模型加载会失败导致启动异常。
> 请确保 `.cache/huggingface/models--BAAI--bge-small-zh-v1.5/` 和 `.cache/huggingface/models--BAAI--bge-reranker-base/` 两个目录存在且包含完整模型文件。

## 第6章 首次运行常见问题

### 6.1 问题：`ModuleNotFoundError` / `ImportError`

**现象：** 后端启动时报找不到某个 Python 包。

**原因：** 依赖未完全安装。

**解决：**
```powershell
cd D:\软件杯\backend
.\.venv\Scripts\activate
pip install -r requirements.txt
```

---

### 6.2 问题：PostgreSQL 连接失败

**现象：** 控制台报错 `could not connect to server` 或 `password authentication failed`。

**原因：** PostgreSQL 未启动 / 密码错误 / 数据库不存在。

**解决：**
1. 确认 PostgreSQL 服务已启动（Windows 服务：`postgresql-x64-16`）
2. 确认 `.env` 中 `DATABASE_URL` 的密码与安装时设定一致
3. 确认已创建数据库 `a5_scenic_guide`

---

### 6.3 问题：Embedding / Reranker 模型加载失败

**现象：** 启动时日志提示模型文件不存在或者 `HF_HUB_OFFLINE=1` 模式下找不到模型。

**原因：** 本地模型缓存缺失。

**解决（二选一）：**

- **方案 A（从其他机器复制）：** 将模型的 `.cache/huggingface/` 整个复制到本机项目根目录
- **方案 B（在线下载）：** 在 `.env` 中临时设置 `HF_HUB_OFFLINE=0`，重启后端让 HuggingFace 自动下载模型（需要 1.5 GB 流量，约 5-15 分钟）

---

### 6.4 问题：`curl` 命令不可用

**现象：** PowerShell 中执行 `curl` 报错或行为异常。

**原因：** Windows PowerShell 中 `curl` 实际上是 `Invoke-WebRequest` 的别名。

**解决：** 使用以下格式替代：
```powershell
Invoke-WebRequest -Uri http://127.0.0.1:8000/api/admin/sync-processed-data -Method POST
Invoke-WebRequest -Uri http://127.0.0.1:8000/api/admin/build-rag-index -Method POST
```
或安装真正的 curl：`winget install curl.curl`

### 6.5 问题：页面空白 / 内容未加载

**现象：** 浏览器打开 `http://localhost:5173/` 后白屏或页面无法显示。

**原因：** 
- 后端未启动或启动失败
- 前端 Vite 代理未正确指向后端
- 浏览器缓存

**解决：**
1. 确认两个 PowerShell 窗口都在运行且无报错
2. 确认 `frontend/vite.config.js` 中 proxy target 指向 `http://127.0.0.1:8000`
3. 按 `Ctrl + Shift + R` 强制刷新浏览器
4. 打开浏览器开发者工具 (F12)，查看 Console 和 Network 面板排查具体错误

### 6.6 问题：DeepSeek / 百炼 API 调用失败

**现象：** 发送问题后无响应，或后端日志显示 API 连接超时 / 认证失败。

**原因：** API Key 无效或网络不通。

**解决：**
1. 检查 `.env` 中 `LLM_API_KEY`、`TTS_API_KEY`、`ASR_API_KEY` 是否正确
2. 确认网络可访问 `https://api.deepseek.com` 和 `https://dashscope.aliyuncs.com`
3. 确认 API Key 有足够余额

### 6.7 问题：端口被占用

**现象：** 后端启动日志显示 `Address already in use`；前端报 `Port 5173 is already in use`。

**原因：** 上一次未正常关闭，或其它程序占用了同一端口。

**解决：**
1. 关闭所有 PowerShell 窗口，重新启动
2. 若仍占用，查找占用进程：
   ```powershell
   # 查看 8000 端口占用
   netstat -ano | findstr :8000
   # 查看 5173 端口占用
   netstat -ano | findstr :5173
   ```
   然后通过任务管理器结束对应 PID 进程

### 6.8 问题：数字人不显示（3D 模型缺失）

**现象：** 页面可以正常问答，但数字人区域为黑屏或显示空白。

**原因：**
- `frontend/public/models/` 下缺少 VRM 模型文件
- 浏览器 WebGL 不可用

**解决：**
1. 确认 `frontend/public/models/` 目录中存在 `.vrm` 文件
2. 在 Chrome 地址栏输入 `chrome://gpu/`，确认 WebGL 状态为 "Hardware accelerated"
3. 若 WebGL 不可用，检查显卡驱动是否正确安装

---

### 6.9 问题：npm install 卡住或报错

**现象：** `npm install` 长时间无响应或报 `ECONNRESET`、`ETIMEDOUT`。

**原因：** npm 默认源访问慢。

**解决：**
```powershell
npm config set registry https://registry.npmmirror.com
npm install
```
