# 修复版前端 5174 CORS 接入计划

## 1. 问题

修复版前端运行在：

```text
http://127.0.0.1:5174
```

后端当前只允许 5173，预检请求返回：

```text
400 Disallowed CORS origin
```

因此聊天请求在浏览器中显示 `Failed to fetch`。

## 2. 修改

仅修改 `backend/.env` 的 `CORS_ORIGINS`，增加：

```text
http://localhost:5174
http://127.0.0.1:5174
```

保留现有 5173 地址。

## 3. 验证

1. 启动后端 8000。
2. 验证健康检查。
3. 分别使用 5173 和 5174 Origin 请求 `/api/chat/stream` 预检。
4. 两个端口均应返回允许的 `Access-Control-Allow-Origin`。
