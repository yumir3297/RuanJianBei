# B3 拍照识景检索线索显示修复计划

## 问题判断

前端“检索线索”区域显示的内容不是字符编码乱码，而是把后端用于 RAG 检索的内部 `retrieval_query` 原样展示给游客。该字段会合并用户问题、图片概述、候选景点、视觉标签和检索提示，因此天然较长、重复度高；同时 Qwen 可能把空 OCR 结果返回为 `[]`，当前后端会把它转成字符串展示，进一步造成“乱码感”。

## 本次修改范围

1. 后端 Qwen 解析层清洗 `detected_text`：空数组、空对象、`null` 等不再进入 `VisionResult.detected_text`，数组文本会归一化为可读字符串。
2. 前端拍照识景卡片不再直接展示内部 `retrieval_query`，改为展示去重后的可读线索标签。
3. “根据图片线索提问”仍继续使用后端生成的 `retrieval_query`，保证检索闭环不变。
4. 不改接口字段、不新增依赖、不更换供应商、不额外调用真实 Qwen API。

## 验证方式

1. 运行后端图片识别相关单元测试，覆盖 Qwen 空 OCR 数组清洗。
2. 运行前端构建，确认 Vue 模板和样式无编译错误。
3. 用户刷新页面后重新识别图片，检索线索区域应显示为短标签，不再出现 `[]` 或大段内部检索串。
