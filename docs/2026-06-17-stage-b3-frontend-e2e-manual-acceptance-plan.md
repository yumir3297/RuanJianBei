# 2026-06-17 阶段 B3.8 前端图片识别端到端人工验收计划

## 背景

B3.7 已完成游客端“拍照识景”入口。当前需要启动本地前后端服务，由用户在页面上手动上传图片，验证前端交互是否能够真实串起：

```text
图片上传
-> /api/vision/analyze
-> Qwen 图片识别结果展示
-> 一键根据图片线索提问
-> 现有聊天/RAG 证据回答
```

## 本轮目标

启动本地后端与前端服务，提供可测试 URL，并给出人工验收步骤。

## 本轮做什么

1. 检查 `8000` 和 `5173` 端口是否被占用。
2. 启动 FastAPI 后端服务。
3. 启动 Vite 前端开发服务。
4. 检查 `/health` 是否可访问。
5. 检查前端页面是否可访问。
6. 给用户本地测试网址和验收步骤。

## 本轮不做什么

1. 不自动上传真实图片。
2. 不自动重复调用 Qwen。
3. 不保存游客上传图片。
4. 不修改数据库 Schema。
5. 不改模型供应商。

## 人工验收重点

1. 页面显示“拍照识景”区域。
2. 选择图片后有预览。
3. 点击“识别图片”后能显示候选景点、视觉标签、置信度和检索线索。
4. 点击“根据图片线索提问”后，聊天区返回答案。
5. 答案包含 `[证据N]`，右侧来源区显示官方资料来源。

## 风险控制

1. 如果端口被占用，先报告占用情况，不强杀未知进程。
2. 如果后端启动失败，记录日志路径。
3. 如果前端启动失败，记录日志路径。
4. 如果页面可访问但接口失败，再针对接口日志修复。
