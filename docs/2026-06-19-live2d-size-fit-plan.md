# Live2D 尺寸与定位修复计划

## 1. 问题

当前 Live2D 使用固定参数：

```text
画布：300 x 400
模型缩放：0.6
锚点：底部居中
位置：x=150, y=360
```

Haru 模型原始尺寸较大，固定缩放后超出画布，页面只显示被放大的腿部。

## 2. 修复内容

1. 将聊天页数字人容器调整为紧凑尺寸。
2. 模型加载完成后读取实际 `model.width/model.height`。
3. 根据画布宽高计算等比缩放值，不再写死 `0.6`。
4. 使用中心锚点，将完整模型居中放入画布。
5. 保留现有表情、口型和 SVG 降级逻辑。

## 3. 不做

1. 不更换模型。
2. 不修改后端或 SSE。
3. 不修改数字人依赖。
4. 不接真实 TTS。

## 4. 验收标准

1. 页面不再只显示腿部。
2. 完整模型位于游客端标题左侧。
3. 模型不溢出聊天卡片。
4. 桌面和窄屏布局保持可用。
5. 前端生产构建通过。
