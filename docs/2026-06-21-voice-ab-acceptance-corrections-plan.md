# 语音 A/B 验收问题修正计划

## 目标

依据 `设计文档/语音AB方案验收报告.md` 修复后端 TTS、浏览器 SpeechSynthesis、前端播放队列与 Live2D 口型之间的实际断点。

## 修正项

1. SSE `audio` 事件补充对应句子文本，供浏览器语音降级使用。
2. 音频队列同时保存 `text` 与 `onProgress`，真实音频解码失败后调用 SpeechSynthesis。
3. SpeechSynthesis 监听 `voiceschanged` 与 `boundary`，并提供定时进度兜底。
4. Live2D 在外部语音进度生效时停用内部固定口型动画，避免互相覆盖。
5. 聊天页统计待播放语音段，避免 SSE 已结束但语音仍播放时提前切回待机。
6. 百炼 TTS 兼容 base64、data URI、音频 URL 和对象响应，并增加可配置超时。
7. 补充后端事件契约与百炼响应解析测试。

## 验收

- Stub TTS 无效音频可降级为浏览器中文播报。
- 真实音频和浏览器播报均持续回传 0 到 1 的进度。
- 播放期间 Live2D 仅由外部进度驱动口型。
- 最后一段音频结束后数字人回到待机。
- 后端专项测试、完整回归和前端生产构建通过。
