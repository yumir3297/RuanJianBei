## Part 2: 系统架构与核心技术

### 第4页：整体技术架构图
- 一张六层架构全景图
- 输入层（文本/语音/图片/主动选择）→ 选择解析 → FAQ快速通道 → Chroma RAG → 多模态LLM → 流式输出（SSE/TTS/Avatar）
- 颜色区分六个阶段

### 第5页：RAG 知识库检索链路
- FAQ L1/L2/L3 三级快速通道（准确率 100%，P95 延迟 13.51ms）
- ChromaDB 向量检索（Recall@1 = 96.67%，Recall@5 = 100%）
- 条件式 CrossEncoder 重排（Recall@1 = 100%，MRR = 100%）
- 关键优化：Reranker P95 从 6079ms 降至 833ms（下降 86%）
- 条件式跳过策略：单候选/精确景点范围内安全跳过

### 第6页：多模态大模型接入
- DeepSeek v4-pro：文本流式问答，证据约束 Prompt（必须用[证据N]引用）
- 阿里云百炼 Qwen-VL-Plus：图片景点识别，输出候选映射为 SelectionContext
- 设计原则：图片识别结果只作为检索线索，不作为事实来源
- 关键数据：综合事实准确率 ≥ 90%，证据引用率 100%，无证据拒答率 100%

### 第7页：主动选择与上下文管理
- SelectionContext 结构图（mode / attraction_id / topic_key / route_id）
- 冲突消解优先级：文字实体 > 主动点击 > 历史上下文 > 默认范围
- 四级引导选择：目标 → 景点 → 话题 → 路线
- 关键数据：5 轮连续追问实体保持率 100%

### 第8页：AI 数字人闭环
- 架构图：ASR → QA Pipeline → TTS → Avatar 状态机
- 六种状态：idle / listening / thinking / speaking / happy / apology
- 情感分析驱动表情：正面 → 微笑，负面 → 关切，感谢 → 点头
- 降级策略：Avatar 故障不阻塞文本与音频回答

### 第9页：知识盲区闭环
- 流程图：无证据问题 → 自动聚合去重 → 管理后台查询 → 补录FAQ → 自动刷新索引
- 关键数据：盲区拒绝率 100%，FAQ L3 正例准确率 100%
- 官方资料同步保护已解决盲区引用的人工 FAQ

## Part 5: 创新亮点

### 第17页：创新点一 — 五级检索通道 + 知识盲区闭环
- FAQ L1/L2/L3 + 选择感知缓存 + Chroma RAG 五级漏斗
- 盲区自动聚合去重 → 管理员补录 → 索引自刷新 → 审计保留

### 第18页：创新点二 — GPS 弱信号混合定位方案
- 问题：灵山梵宫、五印坛城等室内景点 GPS 不可用
- 方案：GPS（室外）→ WiFi指纹（室内）→ 手动选择（兜底）三级降级
- 预留 LocationProvider 抽象接口，兼容未来扩展

### 第19页：创新点三 — 情感分析驱动的数字人表情联动
- 游客对话实时情感打分 → 正面微笑 / 负面关切 / 感谢点头
- 后台情感趋势报告：近 7 天曲线 + Top 5 低分对话定位
- 设计层面预留面部表情识别扩展
