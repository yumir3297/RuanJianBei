# Live2D 尺寸与定位修复报告

## 1. 问题原因

Haru 模型使用固定 `scale=0.6`、底部锚点和固定坐标，模型实际尺寸远大于画布，所以页面只显示被放大的腿部。

## 2. 已完成修改

1. Live2D 画布调整为 `180 x 220`。
2. 窄屏显示区域调整为 `140 x 172`。
3. 加载后先将模型缩放恢复为 `1`。
4. 根据模型实际宽高和画布可用空间计算等比 `contain` 缩放。
5. 锚点改为模型中心 `0.5, 0.5`。
6. 模型位置改为画布正中心。
7. 保留 Cubism 4 入口、Pixi 6.5.10、表情、口型和 SVG 降级。
