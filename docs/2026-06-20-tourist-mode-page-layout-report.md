# 游客端模式选择拆页与问答布局优化实施报告

## 完成内容

1. 新增 `/tourist/select` 独立问答模式选择页，首页游客入口改为先进入该页面。
2. 四种问答模式改为桌面端 2×2 卡片布局，继续读取现有景点、话题和路线数据。
3. 新增选择确认机制；未确认离开选择页时恢复原模式，避免误改当前问答上下文。
4. 聊天页移除大型 `GoalSelector`，新增“更换问答模式”、当前模式面包屑和问答状态。
5. 聊天主体调整为左侧数字人约 32%、右侧对话约 68%。
6. 拍照识景改为按需展开，发送按钮作为唯一主操作，语音与图片作为次级工具。
7. 资料来源从固定悬浮抽屉改为聊天容器内部底栏，不再遮挡输入区域。
8. 切换到不同模式时保留历史消息，但清空旧会话上下文、快捷追问和资料来源。
9. 路由切换统一回到页面顶部，避免继承上一页滚动位置。

## 保持不变

1. 后端 API、数据库 Schema、SSE 事件和问答主链路未修改。
2. `SelectionContext` 字段和请求结构未修改。
3. 图片识别、语音输入、数字人、引用定位和文本降级逻辑未修改。
4. 本轮仅验收桌面端，不新增手机端设计。

## 验证结果

1. `npm.cmd run build` 通过。
2. `1366×768` 下无横向溢出，模式卡片为 2×2，问答页底部来源栏可见。
3. `1920×1080` 下选择页和问答页布局完整，左右栏比例符合方案。
4. 景点模式未选择具体景点时，确认按钮保持禁用。
5. 从问答页返回选择页后，未确认的新选择不会覆盖原模式。
6. 页面浏览器控制台未发现运行错误。

## 已知提示

Vite 构建仍会报告项目原有的大 chunk 警告和第三方 `@vueuse/core` 注释提示，本轮未新增相关依赖，也未扩大处理范围。

## 后续界面精简

1. 删除模式选择页底部“准备进入”确认栏，四张模式卡片改为点击后直接进入问答页。
2. 顶栏品牌区删除印章、原名称和副标题，仅保留“数字人智能导览系统”。
3. 删除数字人降级组件中的“待机中、聆听中、思考中、播报中”等可见状态文字。
4. 保留数字人内部状态机、口型、表情和动画联动，避免削弱赛题要求的数字人自然度能力。
5. 修正 Live2D 默认模型：由不兼容 Cubism 4 入口的 Shizuku `model.json` 改为 Haru `model3.json`。
6. 前端生产构建通过，浏览器验证卡片可直接跳转且页面不再显示数字人状态标签。
