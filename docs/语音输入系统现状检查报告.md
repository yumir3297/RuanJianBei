## 一、系统架构

```
┌──────────────────────────────────────────────┐
│  前端 ChatView.vue （Voice-First 布局）        │
│                                                │
│  主入口：大圆形麦克风按钮（按住说话，松手发送）     │
│  降级入口：可折叠文字输入                          │
│                                                │
│  录音链路：useRecorder.js                       │
│    → navigator.mediaDevices.getUserMedia       │
│    → MediaRecorder（webm/opus, 100ms切片）      │
│    → Blob → POST /api/asr/transcribe           │
│                                                │
│  降级链路：useSpeechRecognition.js              │
│    → window.SpeechRecognition（Chrome 内置）    │
│    → Google 云端识别（需要联网，仅 Chrome）       │
└──────────────┬───────────────────────────────┘
               │ POST /api/asr/transcribe
               ▼
┌──────────────────────────────────────────────┐
│  backend/app/api/voice.py                     │
│                                                │
│  if settings.asr_provider == "bailian":        │
│    → BailianASRService(settings)              │
│  else:                                         │
│    → StubASRService()  ← 当前走这条路          │
└──────────────┬───────────────────────────────┘
               ▼
┌──────────────────────────────────────────────┐
│  BailianASRService（224行，代码已就绪）           │
│                                                │
│  1. 音频存本地 → 生成公网URL                     │
│  2. POST 百炼提交任务（paraformer-v2）           │
│  3. 轮询任务状态 → Succeeded                   │
│  4. 提取转写文本 → ASRResult                   │
│                                                │
│  返回：text, confidence=0.78(heuristic),       │
│        provider="bailian", candidates          │
└──────────────────────────────────────────────┘
```

## 三、关键缺口

### 唯一缺口：.env 未配置 ASR

`.env.example` 第 43–57 行有完整的 ASR 配置段：

```
ASR_PROVIDER=bailian
ASR_API_KEY=
ASR_MODEL=paraformer-v2
ASR_BASE_URL=https://dashscope.aliyuncs.com/api/v1/services/audio/asr/transcription
ASR_TASK_URL=https://dashscope.aliyuncs.com/api/v1/tasks
ASR_VOCABULARY_ID=
ASR_PUBLIC_AUDIO_BASE_URL=
ASR_UPLOAD_DIR=../.uploads/asr
ASR_TIMEOUT_SECONDS=30
ASR_POLL_INTERVAL_SECONDS=0.8
ASR_MAX_AUDIO_BYTES=10485760
```

**实际 `.env` 中无任何 ASR 配置行**。因此运行时 `config.asr_provider` 走默认值 `"stub"`，`voice.py` 第 18 行判断 `!= "bailian"`，跳过 BailianASRService，所有录音一律走 `StubASRService`，返回 `"语音输入占位文本"`。

### 要启用真实 ASR，在 `.env` 中追加

```
ASR_PROVIDER=bailian
ASR_API_KEY=<你的百炼API Key，和TTS用同一个即可>
ASR_PUBLIC_AUDIO_BASE_URL=http://<你的局域网IP>:8000
```

`ASR_PUBLIC_AUDIO_BASE_URL` 必须填一个百炼服务器能访问到的地址（演示时用手机热点或局域网 IP + 端口，百炼通过这个 URL 拉取音频文件做识别）。
