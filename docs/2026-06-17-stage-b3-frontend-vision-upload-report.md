# 2026-06-17 阶段 B3.7 前端图片上传入口实施报告

## 本轮目标

将 B3.6 已验证成功的图片识别与 RAG 闭环能力接入游客端页面，使演示时可以从前端上传图片、查看 Qwen 识别线索，并一键进入现有聊天问答链路。

## 已完成内容

1. 新增前端 API：`frontend/src/api/vision.js`。
2. 在游客聊天页 `frontend/src/views/tourist/ChatView.vue` 增加“拍照识景”区域。
3. 支持选择本地图片，限制单张图片不超过 `5MB`。
4. 支持图片预览、文件名和大小展示。
5. 支持调用 `POST /api/vision/analyze`。
6. 支持展示：
   - 场景描述
   - 候选景点
   - 视觉标签
   - 置信度
   - Provider
   - `retrieval_query`
7. 支持“一键根据图片线索提问”，将 `retrieval_query` 送入现有聊天流。
8. 更新后端视觉默认超时为 B3.5 已验证成功的范围：
   - `VISION_READ_TIMEOUT_SECONDS=60`
   - `VISION_TOTAL_TIMEOUT_SECONDS=90`
9. 更新 `backend/.env.example`，补充视觉超时配置示例。

## 关键设计

前端文案明确提示：

```text
Qwen 识别图片，只作为检索线索；最终答案仍以官方资料证据为准。
```

一键提问会构造如下查询：

```text
请根据这张图片识别出的线索，基于官方资料介绍它最可能对应的景区景点。
图片识别线索如下，仅用于检索官方资料，不作为事实来源：
{retrieval_query}
```

为避免超过后端 `ChatRequest.query` 长度限制，前端会将最终提问裁剪到安全长度内。
