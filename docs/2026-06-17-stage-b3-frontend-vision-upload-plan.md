# 2026-06-17 阶段 B3.7 前端图片上传入口执行计划

## 背景

B3.6 已完成后端最小闭环：

```text
图片 -> Qwen 图片识别 -> retrieval_query -> 官方资料 RAG/FAQ -> 带 [证据N] 的回答
```

当前缺少游客端可操作入口。B3.7 的目标是让演示页面可以上传图片、展示识别线索，并一键将图片线索送入现有聊天问答。

## 本轮目标

在游客聊天页增加图片识别交互：

1. 游客选择本地图片。
2. 前端调用 `POST /api/vision/analyze`。
3. 页面展示 Qwen 识别结果。
4. 游客点击“一键根据图片提问”。
5. `retrieval_query` 进入现有聊天流。
6. 最终回答仍由官方资料证据链负责。

## 本轮做什么

1. 新增 `frontend/src/api/vision.js`。
2. 在 `ChatView.vue` 中增加图片上传区域。
3. 展示：
   - 图片文件名
   - 场景描述
   - 候选景点
   - 视觉标签
   - 置信度
   - 检索线索
4. 增加“一键根据图片提问”按钮。
5. 视觉接口单独设置较长前端超时，避免 Qwen 图片识别 20 秒以上时被 axios 默认 10 秒中断。
6. 后端视觉默认超时调整为与 B3.5 成功烟测一致的安全值。
7. 执行前端生产构建、后端回归和编译检查。

## 本轮不做什么

1. 不主动上传真实图片调用 Qwen。
2. 不保存游客原图。
3. 不新增数据库表。
4. 不修改现有聊天 SSE 协议。
5. 不让 Qwen 直接回答景区事实。
6. 不批量测试图片。

## 成功标准

1. 前端构建通过。
2. 后端回归通过。
3. 游客端有图片上传入口。
4. 识别结果可展示。
5. `retrieval_query` 可一键进入现有聊天/RAG 流程。
6. 文案明确提示：图片识别结果仅作为检索线索，最终回答以官方资料证据为准。
