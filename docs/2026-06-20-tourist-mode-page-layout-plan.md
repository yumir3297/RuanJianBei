# 游客端模式选择拆页与问答布局优化计划

## 目标

1. 点击首页“游客端”后，先进入独立问答模式选择页。
2. 将聊天页顶部的大型模式选择区域移除，释放对话空间。
3. 在聊天页提供“更换问答模式”入口，并明确展示当前选择。
4. 重新整理发送、语音、拍照识景和资料来源区域，降低按钮拥挤感。

## 约束

1. 仅适配桌面端，验收分辨率为 1366×768 和 1920×1080。
2. 不修改后端 API、数据库、SSE 事件、RAG、语音和图片识别业务逻辑。
3. 继续使用结构化 `SelectionContext`，景点、话题和路线仍来自现有 bootstrap API。
4. 继续通过 `localStorage` 恢复已确认的问答模式。
5. 切换模式后保留页面中的历史消息，但清空旧会话上下文、快捷追问和资料来源，避免语境串线。
6. 数字人、语音或图片能力失败时，文本问答仍须可用。

## 实施步骤

1. 新增 `/tourist/select` 路由和独立模式选择页面，首页游客入口指向新页面。
2. 为模式选择增加确认操作；离开页面但未确认时恢复进入页面前的选择。
3. 聊天页移除 `GoalSelector`，增加返回选择页按钮、当前模式面包屑和问答状态。
4. 将聊天主体调整为左侧数字人约 32%、右侧对话约 68%。
5. 将拍照识景改为按需展开的工具区，发送按钮作为唯一主操作。
6. 将资料来源抽屉改为聊天容器内部的底部区域，避免固定定位遮挡输入区。
7. 运行前端生产构建，并在桌面浏览器检查页面流转、模式切换和主要布局。

## 验收标准

1. 首页进入游客端后首先看到独立模式选择页。
2. 四种模式以宽松的 2×2 卡片展示，景点、话题和路线选择仍能加载真实数据。
3. 确认后进入问答页，当前模式显示正确。
4. “更换问答模式”可返回选择页；切换后旧服务端上下文被清除。
5. 聊天页不再出现顶部大型模式选择模块，输入工具和资料来源不遮挡对话。
6. `npm run build` 通过，1366×768 和 1920×1080 下无明显横向溢出或按钮重叠。
