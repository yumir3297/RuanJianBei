# 语音 A/B 验收问题修正报告

## 结果

语音输出已形成统一的三级链路：

```text
百炼真实音频
  -> Web Audio 解码播放
  -> 解码失败时 SpeechSynthesis 中文播报
  -> 浏览器不支持时静默降级，文本问答保持可用
```

真实音频和 SpeechSynthesis 现在共用同一套播放进度回调，并由 `ChatView` 传给 Live2D。外部进度生效时，Live2D 的内部固定口型定时器会暂停，不再覆盖同步结果。

## 主要改动

1. SSE 音频事件增加原句文本和口型时间线。
2. 音频队列恢复 SpeechSynthesis 降级所需的文本参数。
3. SpeechSynthesis 增加中文音色异步加载、边界进度和定时进度兜底。
4. 聊天页追踪待播放语音段，避免 SSE 先结束导致数字人提前待机。
5. 最后一段播报结束后立即关闭口型。
6. 百炼响应兼容多种音频载荷格式，并支持请求超时配置。
7. 新增百炼响应解析与 SSE 音频事件契约测试。

## 验证边界

内置自动化浏览器未提供 SpeechSynthesis API，本轮没有把方案 B 的真实声音听测标记为自动化通过。桌面 Chrome/Edge 仍需完成一次人工听测。

百炼真实网络合成也未在自动化回归中调用，避免消耗线上额度；代码通过模拟的 base64、data URI、URL 和对象响应进行验证。
