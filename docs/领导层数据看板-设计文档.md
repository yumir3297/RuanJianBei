---

### API 1：`GET /api/insights/spot-attention`

**功能**：景点关注度排名。从 `ChatLog.sources` 中反解析被引用的景点名称，统计每个景点被提及的次数。

**实现思路**：
- `select(ChatLog.sources)` 取出所有问答日志的 sources JSON
- 对每个 sources 列表提取其中的 `title` 字段
- 按 title 分组计数，取 Top 10 返回

**响应**：

```json
[
  { "name": "灵山大佛",   "count": 523, "ratio": 0.42 },
  { "name": "梵宫",       "count": 349, "ratio": 0.28 },
  { "name": "九龙灌浴",   "count": 224, "ratio": 0.18 },
  { "name": "五印坛城",   "count": 100, "ratio": 0.08 },
  { "name": "祥符禅寺",   "count": 50,  "ratio": 0.04 }
]
```

**Schemas**（`backend/app/schemas/insights.py`）：

```python
from pydantic import BaseModel

class SpotAttentionItem(BaseModel):
    name: str
    count: int
    ratio: float
```

---

### API 2：`GET /api/insights/visitor-groups`

**功能**：游客群体画像。统计 `VisitorProfile` 中 `audience_type` 的分布，和 `preference_tags` 的热门标签排名。

**实现思路**：
- `SELECT audience_type, COUNT(*) FROM visitor_profiles GROUP BY audience_type`
- `SELECT preference_tags` 全量读取后解析 JSON，合并统计标签频次，取 Top 10

**响应**：

```json
{
  "audience_distribution": [
    { "type": "亲子家庭", "count": 34, "ratio": 0.38 },
    { "type": "普通游客", "count": 25, "ratio": 0.28 },
    { "type": "情侣游客", "count": 14, "ratio": 0.16 }
  ],
  "top_tags": [
    { "tag": "自然风光", "count": 45 },
    { "tag": "历史文化", "count": 38 },
    { "tag": "佛教文化", "count": 31 }
  ]
}
```

**Schemas**：

```python
class VisitorGroupItem(BaseModel):
    group_label: str
    count: int
    ratio: float

class TagItem(BaseModel):
    tag: str
    count: int

class VisitorGroups(BaseModel):
    audience_distribution: list[VisitorGroupItem]
    top_tags: list[TagItem]
```

### API 3：`GET /api/insights/qa-trend`

**功能**：问答量趋势 + 热点分布。

- 趋势：按 `ChatLog.created_at` 每天汇总计数，最近 30 天
- 热点类别：按 `ChatLog.hit_level` 分组（FAQ 命中、RAG 生成、缓存命中、盲区）

**实现思路**—趋势：
- `SELECT DATE(created_at), COUNT(*) FROM chat_logs WHERE created_at >= 30天前 GROUP BY DATE(created_at) ORDER BY DATE(created_at)`

**实现思路**—热点分布：
- `SELECT hit_level, COUNT(*) FROM chat_logs GROUP BY hit_level`

**响应**：

```json
{
  "daily_trend": [
    { "date": "2026-06-01", "count": 42 },
    { "date": "2026-06-02", "count": 58 }
  ],
  "hit_distribution": [
    { "level": "faq", "label": "FAQ 精确匹配", "count": 680, "ratio": 0.55 },
    { "level": "rag",  "label": "RAG 知识检索", "count": 460, "ratio": 0.37 }
  ]
}
```

**Schemas**：

```python
class DailyTrendItem(BaseModel):
    date: str
    count: int

class HitDistributionItem(BaseModel):
    level: str
    label: str
    count: int
    ratio: float

class QATrend(BaseModel):
    daily_trend: list[DailyTrendItem]
    hit_distribution: list[HitDistributionItem]
```

### API 4：`GET /api/insights/blind-spot-top`

**功能**：知识库盲区 Top 10。已有 `GET /api/admin/blind-spots`，这个是对游客的展示版本——只取 open 状态、Top 10、去掉内部 ID 字段。

**响应**：

```json
[
  { "rank": 1, "query": "九龙灌浴门票多少钱",      "hit_count": 12 },
  { "rank": 2, "query": "附近有什么好吃的",          "hit_count": 8  },
  { "rank": 3, "query": "景区有寄存行李的地方吗",     "hit_count": 6  }
]
```

**Schemas**：

```python
class BlindSpotTopItem(BaseModel):
    rank: int
    query: str
    hit_count: int
```

## 四、前端：重写 AnalyticsReport.vue

### 依赖

`echarts` 已在项目中安装（^6.1.0），前端可以引用。

### API 调用

```js
// frontend/src/api/insights.js — 新建
import { http } from "./http";

export function fetchSpotAttention() {
  return http.get("/api/insights/spot-attention").then((r) => r.data);
}
export function fetchVisitorGroups() {
  return http.get("/api/insights/visitor-groups").then((r) => r.data);
}
export function fetchQATrend() {
  return http.get("/api/insights/qa-trend").then((r) => r.data);
}
export function fetchBlindSpotTop() {
  return http.get("/api/insights/blind-spot-top").then((r) => r.data);
}
```

### 改造方案

删掉现有 AnalyticsReport.vue 的全部内容，新页面结构如下：


**图表类型**：
- 景点热度 → ECharts 横向柱状图（`type: "bar"`, `xAxis` 和 `yAxis` 互换）
- 游客群体 → 饼图（`type: "pie"`, `radius: ["40%", "70%"]`）
- 问答趋势 → 折线图（`type: "line"`, `smooth: true`）
- 热点分布 → 环形饼图（`type: "pie"`）

**每个图表在 `onMounted` 后 `await` 对应 API 数据，然后 `setOption`。**
